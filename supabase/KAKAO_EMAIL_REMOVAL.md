# 카카오 로그인 account_email 제거 가이드

카카오 로그인 시 `account_email` 항목을 제거하는 방법은 두 가지가 있습니다:

## 방법 1: 카카오 디벨로퍼스 콘솔에서 설정 (권장)

이 방법이 가장 확실하고 권장되는 방법입니다.

### 단계별 가이드

1. **카카오 디벨로퍼스 콘솔 접속**
   - [카카오 디벨로퍼스](https://developers.kakao.com/)에 로그인

2. **애플리케이션 선택**
   - 상단 메뉴에서 '내 애플리케이션' 클릭
   - 설정을 변경할 애플리케이션 선택

3. **카카오 로그인 설정 이동**
   - 왼쪽 사이드바에서 '제품 설정' > '카카오 로그인' 선택

4. **동의 항목 관리**
   - '동의 항목' 탭 클릭
   - 현재 설정된 동의 항목 목록 확인

5. **account_email 항목 비활성화**
   - 목록에서 `account_email` 항목 찾기
   - 해당 항목이 '필수 동의'로 설정되어 있다면:
     - 먼저 '선택 동의'로 변경
     - 그 다음 비활성화 또는 삭제
   - '선택 동의'로 설정되어 있다면 바로 비활성화 가능

6. **설정 저장**
   - 변경 사항 저장하여 적용

### 주의사항

- 기존에 로그인한 사용자의 경우, 이미 동의한 정보는 유지됩니다
- 새로운 사용자부터는 이메일 동의 요청이 표시되지 않습니다
- 필수 동의 항목은 삭제할 수 없으므로, 먼저 선택 동의로 변경해야 합니다

## 방법 2: 코드 레벨에서 제어 (제한적)

Supabase는 내부적으로 OAuth 플로우를 처리하므로, 코드에서 직접 scope를 제어하기는 어렵습니다. 

하지만 Supabase Dashboard에서도 일부 설정이 가능할 수 있습니다:

1. **Supabase Dashboard 접속**
   - 프로젝트 선택
   - Authentication > Providers > Kakao 이동

2. **OAuth 설정 확인**
   - 현재 설정된 OAuth 옵션 확인
   - 가능한 경우 scope 관련 설정 확인

## 확인 방법

설정이 제대로 적용되었는지 확인하려면:

1. **새로운 사용자로 테스트**
   - 기존에 로그인하지 않은 브라우저/계정으로 테스트
   - 카카오 로그인 시 이메일 동의 화면이 나타나지 않는지 확인

2. **카카오 디벨로퍼스 콘솔 확인**
   - 동의 항목 목록에서 `account_email`이 비활성화되어 있는지 확인

## 추가 정보

- 카카오 로그인 기본 scope: `profile_nickname` (이메일 제외)
- 이메일이 필요한 경우에만 `account_email` scope를 요청해야 합니다
- 이메일을 제거하면 사용자 경험이 개선되고, 개인정보 보호에도 도움이 됩니다
